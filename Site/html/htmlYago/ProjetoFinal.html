Projeto Final

Este projeto foi realizado para a matéria de Design de Software, foi o último projeto do semestre dessa matéria. Foram formados grupos de três pessoas e o objetivo era criar um aplicativo que interagisse com o usuário. No meu caso, foi feito um programa de contador de macro nutrientes durante o dia, que fornecia a quantidade de macro nutrientes que o usuário deveria ingerir durante seu dia, a partir do seu objetivo estético e suas informações pessoais.

Materiais utilizados

1 Computador

Detalhes(inicio, desenvolvimento, conclusão)

O Projeto Final de Design de Software foi o último projeto do semestre que se iniciou no meio de maio e teve duração de aproximadamente um mês. O objetivo do projeto era criar um aplicativo que interagisse com o usuário, podendo ser um jogo, uma rede social, um aplicativo de uso pessoal, etc. O aplicativo tinha que ser obrigatoriamente feito com codificação Python. Nos primeiros dias de trabalho, cada grupo desprendeu algumas ideias. Depois de escolhida a ideia principal, alguns sketches foram feitos das interfaces do programa. O restante do trabalho consistiu em codificação, até que no final, com o trabalho pronto e funcional, os projetos foram apresentados aos professores e aos alunos.

O programa de contagem de macro nutrientes foi feito com o “tkinter”. Ao abrir o programa, o usuário teria que escolher o seu objetivo, sendo eles ganhar massa ou perder peso. Independente do objetivo escolhido, o usuário seria direcionado à uma página na qual seria preenchida com as suas informações pessoais, sendo elas peso, altura, idade e nível de atividade física. A partir dos dados preenchidos, o programa calcularia a quantidade de carboidratos, gorduras e proteínas que o usuário deveria consumir durante o dia, assim como a quantidade já consumida durante o dia, que, obviamente, sempre começava no zero.

A partir disso, o usuário deveria relatar ao programa os alimentos que ele consumiu no dia, assim como suas respectivas quantidades. Para isso, o programa teve que conter um banco de dados da Anvisa, que continha todos os alimentos registrados por ela. Dessa maneira, se o usuário comesse uma banana, bastava encontrar no banco de dados tal alimento e registrar a quantidade comida. Conforme o usuário fosse adicionando seus alimentos, o programa atualizava os macro nutrientes já consumidos durante o dia.

Ao final do dia, o usuário deveria clicar na opção “Finalizar dia”, que reiniciaria o programa.

Por trás de tudo isso, houve diversos códigos. Para a criação desse programa, foram criados 3 arquivos. O primeiro seria responsável pelos algoritmos do programa, ou seja, utilizaria as informações do usuário para calcular as quantidades de cada nutriente. O segundo seria responsável pela interface gráfica do programa, ou seja, mostraria ao usuário todas as etapas descritas nos parágrafos anteriores. Por fim, o último seria um arquivo responsável por armazenar os alimentos e macro nutrientes consumidos pelo usuário.
